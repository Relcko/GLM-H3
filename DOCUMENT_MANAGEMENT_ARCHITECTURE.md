# Document Management Architecture — Marketplace Investment Engine

**Companion to:** `MARKETPLACE_INVESTMENT_ENGINE.md`, `EVENT_ARCHITECTURE.md`,
`PERMISSION_MODEL.md`, and the Document Vault module (`RELCKO_ECOSYSTEM_ARCHITECTURE.md`
§3.7). Architecture only.

Defines the **document engine** that the investment flow depends on: the artifact
types, their lifecycle, linkage to compliance, and integration with the Document
Vault module. It does not re-specify the Vault's storage/access layer — it
specifies *which* documents the engine produces/requires and *when*.

---

## 1. Document artifact types (engine scope)

| Artifact | When created | Linked to |
|----------|--------------|-----------|
| **Prospectus** | `PropertyApproved` | Property / SPV |
| **SPV Agreement** | SPV formed (`PropertyApproved`) | SPV |
| **Legal Documents** | throughout | Property / SPV |
| **Valuation Reports** | independent valuation | Property (→ Valuation Engine) |
| **Insurance** | `PropertyOperational` | Property / SPV |
| **Government Approvals** | `REVIEW`/`APPROVED` | Property (jurisdiction) |
| **Financial Statements** | periodic | SPV / Property |
| **Investment Certificates** | `InvestmentComplete` | Investor + Investment + Ownership |
| **Tax Documents** | dividend year-end / sale | Investor (→ Tax Ledger) |

All map to the `Documents` entity (entity 14) and live in Document Vault.

---

## 2. Document lifecycle

```
UPLOADED → (CLASSIFY) → UNDER_REVIEW → VERIFIED | REJECTED
                                          │
                                    ACCESS_GRANTED (grantee scoped)
                                          │
                                    ACCESSED (audit-logged)
```

- `DocumentUploaded` → `DocumentVerified` (compliance) → `AccessGranted`.
- PII/KYC docs (`Documents` category KYC) are **never public**; access only via
  `AccessGrant` to Compliance/Admin. This fixes the legacy open-download gap
  (`MIGRATION_STRATEGY.md` Phase E).
- Every access appends `AuditLog` (entity 19).

---

## 3. Compliance document requirements (gating)

Per `MARKETPLACE_INVESTMENT_ENGINE.md` §8, the following must be **VERIFIED**
before an investment can reach `COMPLIANCE_REVIEW → APPROVED`:

- **KYC** (entity 13): identity of investor (retail/corporate/institutional).
- **AML**: screening on source of funds.
- **PEP**: politically exposed person check.
- **Sanctions**: OFAC/UN/EU list screening.
- **Source of Funds**: proof for amounts above threshold.
- **Risk Scoring**: composite score → auto-pass / manual-review.
- **Manual Review**: human sign-off for flagged/high-risk.
- **Document Verification**: prospectus/SPV/approvals present + authentic.
- **Audit Trail**: full `AuditLog` chain for the investment.

A property cannot move `REVIEW → APPROVED` without the compliance document set;
an investment cannot move `PAID → APPROVED` without the investor's KYC/AML clear.

---

## 4. Document ↔ investment linkage

- `InvestmentCertificate` is generated at `InvestmentComplete` and references the
  `Ownership` record + `Payment` + `ValuationRecord` (cost basis).
- `TaxDocument` is generated by the Tax Ledger at distribution/sale and references
  the relevant `Transaction`s.
- Documents are content-addressed (hash) so tampering is detectable; the hash may
  later be anchored on-chain (future blockchain hook — not implemented here).

---

## 5. Integration with Document Vault module

| Engine → Vault | Vault → Engine |
|---------------|---------------|
| writes artifact records (Prospectus, Certificate, Tax) | verifies + grants access |
| requests `KYC` verification status | emits `KYCApproved`/`KYCRejected` → gates invest |
| links `Documents` to Property/SPV/Ownership | enforces `AccessGrant` scoping |

The engine treats Vault as the **authoritative document store**; it only owns the
*business rules* about which documents are required at which lifecycle stage.

---

## 7. Scalability & safety

- Documents stored in object storage + CDN with **signed URLs** (no open
  download).
- Access grants indexed for fast authorization; cached with invalidation on
  revoke.
- Hashes immutable; versioned artifacts (new version = new `Documents` record).
- PII segregated; compliance docs excluded from any public/cache layer.
- At scale: document metadata in the same partitioned store as the rest of the
  event ecosystem; binary blobs in scalable object storage.
