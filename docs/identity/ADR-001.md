# ADR-001: Identity v2 Domain Architecture

Status: Final

## Context

Identity v2 owns the domain model for users, wallets, passkeys, sessions, and authentication attempts. Sprint 2.1 is stabilization only. Aggregate boundaries, event sourcing, replay, optimistic concurrency, and snapshots remain unchanged.

## Decision

Identity v2 remains a DDD and event-sourced domain with one aggregate per consistency boundary:

- User
- Wallet
- Passkey
- Session
- AuthenticationAttempt

Each aggregate is mutated only through aggregate methods. State transitions are expressed as domain events from the identity event catalog. Rebuilds must be deterministic from event history, and snapshots are state serialization only.

## Stabilization Notes

- Passkeys are not usable until verified and active, and must not be revoked.
- User restore returns users to inactive status and requires explicit activation.
- Repository interfaces are typed to concrete aggregate roots.
- Primary wallet demotion is emitted before revoke or unlink when applicable.
- The event catalog includes only emitted identity events.

## Non-Goals

- No new aggregates.
- No new feature flows.
- No aggregate boundary changes.
- No replay, snapshot, or concurrency redesign.
